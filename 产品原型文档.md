# AI 自律发带 — 产品原型文档

## 一、产品概述

一款佩戴在头部的轻量化设备，通过前额摄像头持续抽帧记录佩戴者第一视角画面，云端 VLM（视觉大模型）判断是否触发用户预设的"不自律行为"，命中后通过头部震动马达即时反馈，形成行为约束闭环。

- **形态**：运动发带 + 前额摄像头模块 + 后脑控制盒
- **重量目标**：<150g
- **续航目标**：≥4 小时（连续工作）
- **制作周期**：2 天

## 三、硬件实施计划

### 阶段 H1：开发板初始化（0.5 天）
1. 给 RDK X3 烧录系统镜像到 SD 卡
2. 串口工具接 RDK 调试，配 WiFi
3. 测试摄像头：`python` 调 OpenCV 拍一帧确认成像正常
4. 测试 GPIO：用 LED 验证一个引脚可控

### 阶段 H2：电路搭建（0.5 天）
1. 锂电池 → 保护板 → 升压模块 → RDK 5V 输入
2. 马达驱动电路：
   - GPIO → 三极管基极（串 1kΩ）
   - 三极管集电极 → 马达 → 5V
   - 马达并联续流二极管（保护）
3. 串接电源开关
4. 全部在面包板/洞洞板上先跑通，再焊接到小块洞洞板

### 阶段 H3：3D 打印外壳（与 H2 并行）
设计两个盒子（用 Snapmaker 自带 Luban 或快速建模）：
- **前额盒**：60×30×20mm，开摄像头孔（直径按摄像头镜头），底部带卡槽穿发带
- **后脑盒**：70×50×25mm，容纳 RDK + 电池 + 驱动板，侧面开 Type-C 充电口、开关孔、马达走线孔
- 用 PETG 打印（韧性比 PLA 好，贴头不易碎）
- 内贴海绵防硌

### 阶段 H4：组装（0.5 天）
1. 摄像头装入前额盒，排线从发带内侧走到后脑盒
2. RDK + 电池 + 驱动板装入后脑盒
3. 马达 2 个：贴在后脑盒内壁（贴头皮位置），震感最强
4. 发带穿过两盒卡槽，调整松紧
5. 上电自检：拍照 + 触发马达 + LED 闪烁

### 阶段 H5：硬件验收
- [ ] 戴上不晃动、不夹头
- [ ] 摄像头视野正对前方
- [ ] 触发震动能明显感知
- [ ] 单次充电运行 ≥3 小时
- [ ] Type-C 可在线充电

## 四、软件实施计划

### 软件架构
```
┌──────────────┐   WiFi/HTTP    ┌─────────────────┐   API   ┌──────────────┐
│  RDK 发带端   │ ────────────►  │ 本地服务（电脑） │ ──────► │  Claude VLM  │
│ - 抽帧       │                │ - 规则引擎       │ ◄────── │              │
│ - 上传       │                │ - 调度任务类规则 │         └──────────────┘
│ - 接收指令    │ ◄────────────  │ - 下发惩罚指令   │
│ - 驱动马达    │                │ - 网页配置面板   │
└──────────────┘                └─────────────────┘
                                         ▲
                                         │ 浏览器
                                         │
                                    [ 用户配置规则 ]
```

### 阶段 S1：发带端固件（RDK Python，0.5 天）
功能：
- 每 N 秒（默认 3s）抽一帧 → JPEG 压缩 → base64 → POST 到本地服务
- 长轮询/SSE 接收服务端指令
- 收到 `{"action":"vibrate","duration":1.0}` → GPIO 拉高 1s
- 自带 LED 心跳指示

关键文件：`headband_client.py`

### 阶段 S2：本地判断服务（Python Flask，0.5 天）
路由：
- `POST /frame`：接收图像 → 调用 Claude Vision → 比对规则 → 返回是否惩罚
- `GET/POST /rules`：规则的读写
- `GET /stream`：实时画面预览
- `POST /punish`：手动触发惩罚（测试用）

调用模型：`claude-opus-4-6`（带视觉），prompt 模板：
```
你是行为审查助手。判断这张第一视角照片中的人是否正在做以下行为之一：
{规则列表}
只回答 JSON：{"matched": ["rule_id1", ...]}
```

### 阶段 S3：规则引擎（0.25 天）
两类规则：
1. **即时违规型**（吸烟、喝酒、刷短视频等）
   - 每帧判断
   - 连续 2 帧命中才触发（防误判）
   - 触发后 30s 冷却
2. **任务窗口型**（1 小时内必须背单词、今天必须健身）
   - 设定时间窗口和达成动作
   - 窗口内任一帧观察到达成 → 标记完成
   - 窗口结束未完成 → 触发惩罚

规则配置示例：
```json
{
  "rules": [
    {
      "id": "no_smoking",
      "type": "instant",
      "description": "吸烟",
      "vlm_check": "画面中是否有人正在吸烟或手持点燃的香烟？",
      "punish": {"vibrate_ms": 1500, "repeat": 2}
    },
    {
      "id": "no_alcohol",
      "type": "instant",
      "description": "饮酒",
      "vlm_check": "画面中是否有人正在喝酒精饮料或出现啤酒/白酒/红酒？",
      "punish": {"vibrate_ms": 1500, "repeat": 2}
    },
    {
      "id": "study_hourly",
      "type": "window",
      "description": "每小时必须背单词",
      "window_minutes": 60,
      "vlm_check": "画面中是否有人在看书、背单词或使用单词 App？",
      "punish": {"vibrate_ms": 2000, "repeat": 3}
    }
  ]
}
```

### 阶段 S4：网页配置面板（0.25 天）
极简单页（HTML + 原生 JS）：
- 实时画面区
- 规则列表（增删改）
- 当前状态：上次抽帧时间、今日触发次数、待完成任务
- 紧急停止按钮

### 阶段 S5：联调测试
- [ ] 端到端：发带拍 → 服务判 → 发带震
- [ ] 故意做违规动作验证命中
- [ ] 故意做正常动作验证不误触
- [ ] 网络断开降级（本地不震，恢复后继续）
- [ ] 紧急停止有效

## 六、风险与备选

| 风险 | 应对 |
|---|---|
| RDK 摄像头排线买不到 | 改 USB 摄像头 |
| 电池续航不足 | 抽帧间隔从 3s 调到 5~10s |
| VLM 判断有延迟（>3s） | 抽帧异步队列，不阻塞主循环 |
| 误判频繁 | 提高连续命中阈值 + 调 prompt |
| 头部震动太弱 | 双马达并联，或换更大功率线性马达 |
| 用户想电击 | **本版本不实现**。涉及安全和监管，需要医疗级方案 |

---

## 七、安全声明

- 仅使用震动反馈，**不实现电击**
- 紧急停止按钮始终可用（硬件开关 + 软件按钮双保险）
- 摄像头数据仅本机/本人电脑处理，不长期存储
- 调用 Claude API 时图片仅用于实时判断，不入库
