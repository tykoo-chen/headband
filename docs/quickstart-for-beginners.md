# 零基础 Git 速成 🐣

## 📖 三个名词，一次说清

### 1. 什么是 Git？
**Git** = 一个"文件版本管理工具"。
比喻：你写论文时会存 `论文.docx`、`论文_最终版.docx`、`论文_最终版_真的最终.docx`、`论文_最终版_真的最终_老板改过.docx` —— Git 就是帮你**自动管理这些版本**的工具，不用再起这种乱七八糟的文件名。

### 2. 什么是 GitHub？
**GitHub** = 一个网站，专门存 Git 项目，方便多人一起改同一个项目。
比喻：Git 是"版本管理软件"，GitHub 是"项目的网盘 + 群聊 + 公告板"。

### 3. 什么是分支（Branch）？
**分支** = 同一份文件的"平行宇宙"。
比喻：你和室友都想改同一份论文，但你们想法不一样：
- 你开一个**分支**叫 `小明的版本`，自己改
- 室友开一个**分支**叫 `室友的版本`，他自己改
- 改完之后，**队长** 看一眼，决定把哪些段落合到正式版（叫 `main` 分支）

这样你俩**互不影响**，最后由队长统一合并。这就是分支。

## 🖥 GitHub Desktop 界面速览

打开 GitHub Desktop 后，记住这 5 个区域：

```
┌────────────────────────────────────────────────────────────┐
│ ① 顶部工具栏：[Current Repository] [Current Branch] [Fetch]│
├──────────────────┬─────────────────────────────────────────┤
│                  │                                          │
│ ② 左侧 Changes   │   ③ 中间 / 右侧 Diff（看你改了啥）       │
│   (改过的文件列表)│                                          │
│                  │                                          │
├──────────────────┴─────────────────────────────────────────┤
│ ④ 左下：Summary（必填）+ Description（选填）               │
│ ⑤ 蓝色按钮：[Commit to <分支名>]                            │
└────────────────────────────────────────────────────────────┘
```

| 区域 | 你要点的地方 |
|---|---|
| ① **Current Branch** | 切换 / 新建分支 |
| ① **Fetch / Pull / Push origin** | 同步代码（拉新 / 推自己的） |
| ② **Changes** 列表 | 看自己改了哪些文件 |
| ④ **Summary** 输入框 | 写一句"我干了啥" |
| ⑤ 蓝色 **Commit** 按钮 | 把改动"保存到本地版本" |

## 🔧 5 个最常用操作

### ① 同步最新代码（每天开工第一件事）

1. 顶部点 **Current Branch** → 切回 `main`
2. 顶部按钮：先点 **Fetch origin**
3. 按钮变成 **Pull origin** → 再点一下
4. ✅ 完成，本地的 `main` 已经是最新

> 💡 **Fetch** = 看一眼远程有没有新东西；**Pull** = 把新东西下载到本地。

---

### ② 新建你自己的分支

1. 顶部点 **Current Branch**
2. 点 **New Branch**
3. **Name** 输入你的分支名（如 `d/headband`）
4. **Based on** 选 `main`
5. 点 **Create Branch**
6. 弹窗问 publish → 点 **Publish Branch** ✅

---

### ③ 提交改动（写完代码 / 文档之后）

1. 确认顶部 **Current Branch** 是你自己的分支（**不要是 `main`！**）
2. 看左侧 **Changes** 列表，列出的就是你改过的文件
3. 左下 **Summary** 写一句话，比如 `E：加 /api/events POST 接口`
4. 点蓝色 **Commit to <分支名>** 按钮
5. ✅ 完成（这一步只是"保存到本地版本"，还没上传到 GitHub）

---

### ④ 推送到 GitHub（Push）

1. 上一步 commit 完后，顶部按钮变成 **Push origin**
2. 点它
3. 等几秒钟，看到按钮变回 **Fetch origin** = 推送成功 ✅

---

### ⑤ 开 PR（Pull Request，让队长合并）

1. 推送完成后，GitHub Desktop 会弹出 **"Create Pull Request"** 按钮 → 点它
2. 浏览器自动跳到 GitHub 网页，标题大概是：**Open a pull request**
3. 网页上确认：
   - **base：** `main`
   - **compare：** 你自己的分支
4. **Title** 写清楚：`[D] 实现摄像头采集模块`
5. **Description** 可写可不写
6. 点绿色按钮 **Create pull request**
7. 🔔 **去群里 @队长 A**：「PR #X 开好了，请合并」
8. ✅ 等队长合并

> 💡 PR 全名 **Pull Request**，意思是"求大佬把我的改动 pull（拉取）到主分支"。

## 🚨 出错了怎么办（最常见的 3 个错误）

### 错误 1：`Failed to push - Updates were rejected`
**意思**：你 push 的时候，远程已经被别人推过新东西了，git 不让你直接覆盖。

**解决**：
1. 顶部点 **Fetch origin**
2. 按钮变成 **Pull origin** → 点
3. 如果没有冲突 → 重新点 **Push origin** ✅
4. 如果出现 **Merge Conflict** → 看下面错误 2

---

### 错误 2：`Merge Conflict`（红色字！）
**意思**：你和别人改了同一个文件的同一行，git 不知道留谁的。

**🛑 第一反应：截图发群里 @队长 A，不要自己乱点！**

如果队长让你自己处理，照这步走：
1. 用 VS Code 打开冲突的文件
2. 找到这种标记：
   ```
   <<<<<<< HEAD
   你写的内容
   =======
   别人写的内容
   >>>>>>> main
   ```
3. **三选一**：
   - 留你自己的 → 删掉 `=======` 到 `>>>>>>>` 之间所有行 + 那 3 行标记
   - 留别人的 → 删掉 `<<<<<<<` 到 `=======` 之间所有行 + 那 3 行标记
   - 两个都要 → 只删掉 `<<<<<<<`、`=======`、`>>>>>>>` 这 3 行标记
4. 保存文件
5. 回 GitHub Desktop → Commit → Push

> ⚠️ 不确定留哪段？**问队长，不要瞎删！**

---

### 错误 3：`Permission denied` / `Repository not found`
**意思**：GitHub 不认识你，或者队长还没把你加成 collaborator。

**解决**：
1. 检查 GitHub Desktop 是不是登的对账号：**File → Options → Accounts**
2. 检查邮箱里有没有队长发的 invitation 邮件 → 点 **Accept invitation**
3. 还不行 → 截图发群里 @队长 A
